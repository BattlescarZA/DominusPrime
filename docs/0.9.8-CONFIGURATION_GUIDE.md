# DominusPrime 0.9.8 - Configuration Guide

## Overview

This guide covers all configuration options for DominusPrime 0.9.8's new features.

## Security Configuration

### Location

Create `config/security.json` in your project root:

```bash
mkdir -p config
touch config/security.json
```

### Security Profiles

The easiest way to configure security is using predefined profiles:

```bash
dominusprime security set-profile <profile>
```

#### Profile Comparison

| Setting | Open | Relaxed | Balanced | Strict | Paranoid |
|---------|------|---------|----------|--------|----------|
| Require Approval | ❌ | ❌ | ✅ | ✅ | ✅ |
| Block Dangerous | ❌ | ❌ | ✅ | ✅ | ✅ |
| Tool Permission | ALLOW | ALLOW | PROMPT | PROMPT | DENY |
| Skills Scanning | ❌ | ✅ | ✅ | ✅ | ✅ |
| Skills Approval | ❌ | ❌ | ❌ | ✅ | ✅ |
| Audit Logging | ✅ | ✅ | ✅ | ✅ | ✅ |
| Rate Limiting | ❌ | ❌ | ✅ | ✅ | ✅ |
| Max Commands/Min | 100 | 50 | 20 | 10 | 5 |

#### When to Use Each Profile

**OPEN** (Development Only)
- Local development
- Trusted environment
- No external users
- Full agent autonomy needed

**RELAXED** (Prototyping)
- Early testing
- Mostly trusted code
- Want some logging
- Minimal interruptions

**BALANCED** (Recommended)
- Production use
- General purpose
- Good security/usability balance
- Most common choice

**STRICT** (High Security)
- Production with sensitive data
- Multi-user environments
- Regulatory compliance
- Extra caution needed

**PARANOID** (Maximum Security)
- Critical infrastructure
- Financial systems
- Maximum control
- Every action reviewed

### Manual Configuration

Create `config/security.json`:

```json
{
  "security_level": "BALANCED",
  
  "shell_require_approval": true,
  "shell_block_dangerous": true,
  
  "tool_default_permission": "PROMPT",
  
  "skills_scan_on_load": true,
  "skills_require_approval": false,
  
  "audit_logging_enabled": true,
  
  "rate_limit_enabled": true,
  "max_commands_per_minute": 20
}
```

### Configuration Options

#### `security_level`

**Type**: string  
**Options**: "OPEN", "RELAXED", "BALANCED", "STRICT", "PARANOID"  
**Default**: "BALANCED"  
**Description**: Base security profile to use

#### `shell_require_approval`

**Type**: boolean  
**Default**: true (for BALANCED)  
**Description**: Require user approval for shell commands

When `true`:
- HIGH and CRITICAL risk commands require approval
- Approval timeout: 120 seconds
- User can approve/reject/timeout

When `false`:
- Only CRITICAL commands are blocked
- HIGH risk commands execute with warning

#### `shell_block_dangerous`

**Type**: boolean  
**Default**: true (for BALANCED)  
**Description**: Automatically block CRITICAL risk commands

When `true`:
- CRITICAL commands are blocked immediately
- No approval option
- Logged as "blocked"

When `false`:
- CRITICAL commands require approval
- User can override (not recommended)

#### `tool_default_permission`

**Type**: string  
**Options**: "ALLOW", "PROMPT", "DENY"  
**Default**: "PROMPT" (for BALANCED)  
**Description**: Default permission level for tools

- **ALLOW**: All tools can execute without asking
- **PROMPT**: Ask before executing each tool
- **DENY**: Block all tools by default

#### `skills_scan_on_load`

**Type**: boolean  
**Default**: true (for BALANCED)  
**Description**: Scan skill files when loading

Checks for:
- Dangerous patterns
- System calls
- File operations
- Network access

#### `skills_require_approval`

**Type**: boolean  
**Default**: false (for BALANCED)  
**Description**: Require approval to load new skills

When `true`:
- User must approve each new skill
- Skill content shown for review
- Scan results displayed

#### `audit_logging_enabled`

**Type**: boolean  
**Default**: true  
**Description**: Enable audit logging

Logs:
- Command executions
- Approvals/rejections
- Security events
- Risk assessments

#### `rate_limit_enabled`

**Type**: boolean  
**Default**: true (for BALANCED)  
**Description**: Enable rate limiting

Prevents:
- Command spam
- Runaway scripts
- Resource exhaustion

#### `max_commands_per_minute`

**Type**: integer  
**Default**: 20 (for BALANCED)  
**Description**: Maximum commands per minute

Limits command frequency:
- Open: 100/min
- Relaxed: 50/min
- Balanced: 20/min
- Strict: 10/min
- Paranoid: 5/min

## Memory Configuration

### Data Directory

Configure where databases are stored:

```python
from pathlib import Path
from dominusprime.database.connection import DatabaseManager

# Default location
db_manager = DatabaseManager(Path("./data"))

# Custom location
db_manager = DatabaseManager(Path("/var/dominusprime/data"))
```

### Experience System

```python
from dominusprime.agents.memory.experience.system import ExperienceSystem

# Requires: chat_model, formatter, and db_manager
system = ExperienceSystem(
    working_dir=Path("./working"),  # Where to store generated skills
    chat_model=your_chat_model,     # ChatModelBase instance
    formatter=your_formatter,        # FormatterBase instance
    db_manager=db_manager            # DatabaseManager instance
)
```

**Parameters:**

- `working_dir`: Directory for generated skill files
- `chat_model`: Language model for analysis and generation (required)
- `formatter`: Message formatter instance (required)
- `db_manager`: Database manager for persistence (required)

### Multimodal System

```python
from dominusprime.agents.memory.multimodal.processor import MediaProcessor
from dominusprime.agents.memory.multimodal.embedder import MultimodalEmbedder

# Media processor
processor = MediaProcessor(
    storage_dir=Path("./data/media")  # Where to store media files
)

# Embedder
embedder = MultimodalEmbedder(
    visual_model="clip-vit-base-patch32",  # CLIP model for images
    text_model="sentence-transformers/all-MiniLM-L6-v2",  # Text embeddings
    audio_model="panns-cnn14"  # Audio embeddings (not yet implemented)
)
```

**Visual Models:**
- `clip-vit-base-patch32` (default) - Fast, good quality
- `clip-vit-large-patch14` - Slower, better quality

**Text Models:**
- `all-MiniLM-L6-v2` (default) - Fast, 384 dimensions
- `all-mpnet-base-v2` - Better quality, 768 dimensions
- `all-distilroberta-v1` - Balanced

### Proactive Delivery

```python
from dominusprime.agents.memory.proactive import (
    ContextMonitor,
    RelevanceScorer,
    DeliveryManager,
    DeliveryStrategy
)

# Context monitor
monitor = ContextMonitor(
    window_size=10,  # Number of recent messages to analyze
    signal_threshold=0.3  # Minimum signal strength to trigger (0.0-1.0)
)

# Relevance scorer
scorer = RelevanceScorer(
    keyword_weight=0.3,  # Weight for keyword matching
    topic_weight=0.25,   # Weight for topic matching
    temporal_weight=0.15,  # Weight for recency
    pattern_weight=0.2,  # Weight for pattern similarity
    context_weight=0.1   # Weight for context similarity
)

# Delivery manager
manager = DeliveryManager(
    context_monitor=monitor,
    relevance_scorer=scorer,
    strategy=DeliveryStrategy.BALANCED,  # AGGRESSIVE, BALANCED, CONSERVATIVE, PASSIVE
    max_deliveries_per_session=10,  # Maximum proactive deliveries
    min_confidence=0.6  # Minimum confidence to deliver
)
```

**Delivery Strategies:**

| Strategy | Min Confidence | Max Deliveries | Use Case |
|----------|---------------|----------------|----------|
| AGGRESSIVE | 0.4 | 20 | Maximum assistance |
| BALANCED | 0.6 | 10 | Normal use |
| CONSERVATIVE | 0.75 | 5 | Minimal interruption |
| PASSIVE | 1.0 | 0 | Disabled |

**Scorer Weights:**

Adjust to prioritize different factors:

```python
# Prioritize recent memories
scorer = RelevanceScorer(
    temporal_weight=0.4,
    keyword_weight=0.25,
    topic_weight=0.2,
    pattern_weight=0.1,
    context_weight=0.05
)

# Prioritize exact keyword matches
scorer = RelevanceScorer(
    keyword_weight=0.5,
    topic_weight=0.2,
    pattern_weight=0.15,
    temporal_weight=0.1,
    context_weight=0.05
)
```

## Database Configuration

### Connection Pooling

```python
# For high-performance scenarios
db_manager = DatabaseManager(
    data_dir=Path("./data"),
    pool_size=5,  # Not yet implemented
    timeout=30.0
)
```

### Database Paths

Default paths:
- `data/experiences.db` - Experience and skills storage
- `data/multimodal.db` - Media items and embeddings
- `data/security.db` - Audit logs and security events

Custom paths:
```python
# Override individual database paths
db_manager.database_paths["experiences"] = Path("/custom/experiences.db")
```

### Database Maintenance

```bash
# View statistics
dominusprime memory stats

# Clean up old data (manual for now)
sqlite3 data/experiences.db "DELETE FROM experiences WHERE created_at < date('now', '-90 days')"
sqlite3 data/security.db "DELETE FROM command_executions WHERE executed_at < date('now', '-90 days')"
```

## Performance Tuning

### Memory Limits

```python
# Limit embedding cache size
embedder = MultimodalEmbedder()
embedder._embedding_cache_size = 100  # Cache last 100 embeddings

# Limit context window
monitor = ContextMonitor(window_size=5)  # Smaller window = less memory
```

### Search Optimization

```python
# Limit search results
results = await index.search(
    query="...",
    limit=5,  # Return fewer results
    similarity_threshold=0.7  # Higher threshold = fewer results
)
```

### Delivery Optimization

```python
# Reduce delivery frequency
manager = DeliveryManager(
    strategy=DeliveryStrategy.CONSERVATIVE,
    max_deliveries_per_session=3,
    min_confidence=0.8
)
```

### Memory

- Use higher confidence thresholds (0.8+) for skill generation

- Keep `window_size` small (5-10) for performance
- Regular database cleanup and archival

### Database

- Use SSD storage for databases
- Regular backups (daily recommended)
- Monitor database size growth
- Archive old audit logs (>90 days)

## Troubleshooting

### High Memory Usage

Reduce:
- `window_size` in ContextMonitor
- Embedding cache size
- Search result limits

### Too Many Deliveries

Adjust:
- `strategy` to CONSERVATIVE or PASSIVE
- Increase `min_confidence`
- Decrease `max_deliveries_per_session`

### Slow Performance

- Use smaller embedding models
- Increase `similarity_threshold`
- Reduce `limit` in searches
- Regular database vacuuming

### Security Too Strict

- Use RELAXED or BALANCED profile
- Adjust `shell_require_approval` to false
- Increase `max_commands_per_minute`
